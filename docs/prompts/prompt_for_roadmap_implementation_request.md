### Prompt for Roadmap Implementation

Carefully review and implement the fixes described in the relevant roadmap document.

### Execution Rules

1. Start implementing following the roadmap recommendations.
2. Fully complete all sub-tasks related to the task before moving forward.
3. After implementation:

   * Verify the changes logically.
   * Ensure no regressions were introduced.
   * Confirm alignment with the documented intent.
   * Check for any compilation errors.
4. Once the a Issue is fully completed:

   * Udate the document.
   * Mark the task and its sub-tasks as **completed**.
5. Stop execution and explicitly ask for permission before proceeding to the next task.

### Technical Constraints

* **Do NOT use scripts, automation tools, or the command shell** to modify code.
* All changes must be implemented through direct code edits only.
* Maintain existing architecture and coding standards unless explicitly instructed otherwise.

### Output Expectations

After completing the task, provide:

* A summary of what was implemented
* Files modified
* Key code changes
* Any assumptions made
* Any risks or edge cases identified
* Confirmation that the document has been updated

Then pause and wait for approval before continuing.
